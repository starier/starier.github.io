Mongodb基础教程

发表于 2016-07-04   |  

安装

简介

Mongodb不同于传统的关系型数据库，是时下最流行的Nosql数据库，使用文件来保存数据，在Mongodb数据库中集合（collection）可对应关系型数据库表的概念，文档（document）对应关系数据库中行的概念，相对于关系型数据库具有查询简单，效率更高等特点，更适应互联网行业的数据管理。

下载

Mongodb数据库在windows下安装非常简单，打开Mongodb官网，点击右上角Download，即可下载本系统的对应版本，本教程使用的是3.2.4版本。下载完成后即可选择安装到本地指定路径中，指定的路径为数据的文件保存位置。

windows服务

Mongodb安装成功后，会自动在windows系统中添加MyMongo服务，且随系统启动而自动启动。因此，安装完成后，可以直接进行Mongodb的操作。

基础操作

将Mongodb的命令添加到path环境变量中，打开cmd命令窗口，使用mongo命令进入数据库操作界面。

数据库操作

查看所有数据库命令：

show dbs

使用/新建数据库：

use 数据库名称

当前正在使用的数据库：

db

CRUD操作

查看所有集合：

show tables

新建集合/表并插入数据：

db.表名.insert({“name”: “admin”})

查看表中数据：

db.表名.find()

Intellij Idea相关的Mongodb插件

当前操作Mongodb数据库主要通过命令行工具，如果使用Intellij Idea进行开发，可以在编辑器中添加Mongodb相关插件，可以方便日常的开发使用。

Vue+webpack+leancloud基础教程（一）

发表于 2016-05-09   |  

简介

在本项目中，Vue主要实现双向数据绑定与路由设计，webpack用于项目打包、发布，页面样式主要使用bootstrap，后台接口采用leancloud实现，用于演示的示例项目适用于所有后台管理系统，为SPA应用，页面由导航、左侧菜单栏、主页面、尾部四部分组成， 并包括登录、注册功能，采用vue的组件化实现方式来实现。

Github地址：vue-example

项目搭建

安装

首先系统必须安装nodejs，推荐安装稳定版。

Vue与webpack的结合浑然天成，在构建大型单页应用时推荐使用webpack作为打包工作，也可以使用Vue的脚手架工具vue-cli来快速构建项目，其中可以包含热加载、单元测试、代码检查等一系列功能。如果未安装该工具可执行以下命令：

npm install -g vue-cli

安装完成后就可以快速生成vue项目并查看初始效果了，执行以下命令生成项目

vue init webpack example

注：如果需要直接查看本项目，执行git clone https://github.com/starier/vue-example.git命令即可获取项目源码。

生成项目后，进入项目中，并执行npm命令下载相应依赖（需要依赖太多，需要执行一段时间）

cd example

npm install

安装完成后，执行以下命令进入开发模式，打开浏览器http://localhost:8080即可看到项目初始效果

npm run dev

项目结构

由Vue脚手架生成的项目，默认目录结构如下

        |-assets
        |-components

    |-test

        main.js  // 首页入口文件
        routers.js  // 路由配置文件
    |-static
    |-test
    |-build
    index.html //首页
    package.json

git本地多个ssh-key连接不同代码服务器的配置

发表于 2016-05-04   |  

在使用git进行日常开发工作时，会出现需要连接多个代码服务器的情况，如工作git连接Oschina、Coding，个人项目连接Github，这时就需要多个ssh-key来分别连接服务器。

配置用户名和邮箱

git全局配置如下

生成ssh-key并指定保存文件名

此时当前路径（推荐在.ssh文件夹路径中执行）会生成一对ssh文件，名称分别为id_rsa和id_rsa.pub

新增并配置config文件

在~/.ssh路径下新建config文件

touch config

使用任一文件编辑器编辑config文件，配置如下：


    #github  (github 配置)

    Host github.com

        User "leshare"

        IdentityFile ~/.ssh/id_rsa_github


    #oschina  (oschina 配置)

    Host git.oschina.net

            User "huzhanjun"

            IdentityFile ~/.ssh/id_rsa

验证是否连接成功

验证Github是否成功建立ssh连接

ssh -T git@github.com

Hi starier! You’ve successfully authenticated, but GitHub does not provide shell
access.

其中starier为你的用户名，显示该信息则证明ssh连接Github成功！

验证oschina是否连接成功

ssh -T git@git.oschina.net
